Arrays vs. Hashes!
Oct 3 '14

Arrays are an ordered collection of objects. These objects can be numbers or strings, but the index has to be an integer. Since arrays are an ordered collection, you can reference the first index by using array[0], and the next index would be array[1]. You increase the index number by 1 each time. Alternatively, you can reference the last index by either array.last or array[-1], and go backwards from -1 to the first index.

Hashes on the other hand have a key=>value pair, and either of these can be a number or string. The indexing is done by the object keys.

The difference between indexing means that in arrays, you can only point to one object. Whereas with hashes, and object key can point towards several value objects. This makes using hashes more beneficial when you have a list that have similar keys. For example:

movies = Hash.new
movies = {
					:Comedy => "The Other Guys, 21 Jump Street",
				:Action => "The Expendables, X-Men"
	}
	puts movies[:Comedy]

This gives us: The Other Guys, 21 Jump Street
